    **OAuth 2.0 user authentication**
      * Build <Term term="private application">private applications</Term> for your organization that requires users to sign in with an <Term term="arcgis account">ArcGIS account</Term>. 
      * Create <Term term="developer credentials">OAuth 2.0 credentials</Term> and implement OAuth 2.0 flows to get an access token.
      * Applications can access all resources available to the signed-in user, including <Term term="location services">location services</Term>, <Term term="spatial analysis services">spatial analysis services</Term>, <Term term="data services">data services</Term>, secure <Term term="item">items</Term>, and <Term term="portal">portal</Term> management operations.
      * <Term term="privileges">Privileges</Term> and portal <Term term="item">item</Term> access are defined by the user's account. 
      * Service usage and data storage is billed to the organization of the user signed into the application.
    
    [Learn more about user authentication](xref://site.documentation/user-authentication)
