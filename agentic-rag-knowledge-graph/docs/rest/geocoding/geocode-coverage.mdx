Various applications can access the ArcGIS Geocoding service, including [ArcGIS Pro](https://pro.arcgis.com/en/pro-app/latest/help/data/geocoding/tutorial-geocode-a-table-of-addresses.htm#ESRI_SECTION1_75BA26148B2C4D48BAE69F3FF18F8CBD), [Map Viewer](https://enterprise.arcgis.com/en/portal/11.3/use/geocode.htm), [Map Viewer Classic](https://enterprise.arcgis.com/en/portal/11.3/use/view-maps.htm), and clients you develop using an ArcGIS Web API, such as [ArcGIS REST API](https://links.esri.com/agol-help/restapi). 

The map and tables in this topic list the countries where there is address 	coverage in the ArcGIS Geocoding service and the languages they support, grouped by relative 	geocoding quality. The overall geocoding quality is a function of 	the degree of street-level address coverage in a country, knowledge 	of a country's address styles, and geocoding performance for 	addresses in that country. The quality levels are subject to change. Typically, the quality of geocoding for a given country improves over time and may be upgraded to a better level, but there may be instances where countries are downgraded to a different level based on user feedback.

Countries where there is no address coverage are classified as Level 4 and are not shaded on the map. Even though there is no address or street-level coverage in these countries, there is still admin-level coverage available. This means that administrative divisions such as cities, counties, and provinces, as well as landmarks in some cases, can be found in them.

Detailed information about open source data used for the ArcGIS Geocoding service is available in the [Esri Geocoding Address Acknowledgements](https://links.esri.com/esri-geocoding-address-acknowledgements) document. 

## Supported country codes

The Supported country codes column in the tables below provides the valid input values for the `sourceCountry`  and `countryCode`  REST API parameters, which can be used with [findAddressCandidates](/find-address-candidates#sourcecountry) , [geocodeAddresses](/geocode-addresses#sourcecountry) , and [suggest](/suggest#sourcecountry)  requests. The values follow the ISO 3166 standard for 2- and 3-character country codes, with a few exceptions. 

In the following cases, there is no official ISO 3166 code for a country or zone: 

-   EUR = European Union

For some countries, multiple country codes are supported. An official ISO 3166 code is assigned to these countries, and they may also be recognized as territories of other countries. The official country code is always presented first in the Supported country codes column in the tables below. All values in the column can be used with the `sourceCountry`  and `countryCode`  REST API parameters. 

The three-character country codes listed in the tables are also returned in the `Country`  (for [findAddressCandidates](/find-address-candidates)  and [geocodeAddresses](/geocode-addresses)  requests) and `CountryCode`  (for [reverseGeocode](/reverse-geocode)  requests) output fields of geocoding responses. 

<Note type={"note"}>

When a geocoded location is within a disputed region, the `Country`  and `CountryCode`  output fields are blank. This includes Kosovo.

</Note>

<Note type={"note"}>

There are additional requirements for geocoding addresses in China. 

To geocode English or Pinyin addresses in China, the [sourceCountry](/geocode-addresses#sourcecountry)  parameter must be included in the request: `sourceCountry=CHN` .

</Note>

<Note type={"note"}>

There are additional requirements for geocoding addresses in South Korea. 

To geocode English or transliterated addresses in South Korea, the [sourceCountry](/geocode-addresses#sourcecountry)  parameter must be included in the request: `sourceCountry=KOR` .

</Note>

## Supported language codes

The Supported language codes column provides the valid values that can be used as input for the `langCode`  parameter, which can be used with the [reverseGeocode](/reverse-geocode#langcode) , [findAddressCandidates](/find-address-candidates#langcode) , and [geocodeAddresses](/geocode-addresses#langcode)  operations. 

The first language code listed in this column is the default language code for the country. The default language code is the value used by default for each country when the `langCode`  parameter is not explicitly passed in a request. This column is primarily applicable to countries where there are multiple supported languages. 

Use the `EN`  language code for returning transliterated or transcribed versions of a language or for returning Pinyin. The Supported languages column indicates the countries where such languages are available.

<Note type={"note"}>

When there are multiple languages listed as supported for a country, it doesn't mean that every address in the country is available in each language. It may be that addresses are available in multiple languages for only one region of the country, or that each language is exclusive to a different region and there is no overlap. The Supported language codes column shows the valid values that can be used for the `langCode`  parameter in a [reverseGeocode](/reverse-geocode#langcode) , [findAddressCandidates](/find-address-candidates#langcode) , or [geocodeAddresses](/geocode-addresses#langcode)  request. The Supported languages column provides the corresponding language names for the codes listed in the Supported language codes column. 

</Note>
