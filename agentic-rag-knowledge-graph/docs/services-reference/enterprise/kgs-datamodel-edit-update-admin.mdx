## Description

<Note type={"note"}>

<GUID_01CB5051_FA79_42A9_8275_EF008F2C3097_GUID_499B73EC_3D4E_46D2_863A_4534FAF79AA3 />

</Note>

The `update`  operation allows an administrative client to modify edit settings associated with the knowledge graph service's [administrative data model resource](/kgs-datamodel-admin). With this operation, you can set the `strict`  property for the knowledge graph service's data model as a whole. For example, making this change would mean only the owner of the service or the service administrator can add or remove entity types and relationship types, and add and remove properties for any type. Instead of setting the entire data model to be strict, you can [change this setting for specific entity types and relationship types](/kgs-datamodel-edit-namedtypes-type-update-admin) that should not be modified.

Use the data model [Query](/kgs-datamodel-query) operation to learn about the item types that are defined in the schema. Use the data model [Named Types](/kgs-datamodel-edit-namedtypes-admin) resource with the name of a specific entity type or relationship type to access information about and modify settings for that graph item.

## JSON Response syntax

```json
{
  "strict": <boolean>
}
```

 

## JSON Response example

```json
{
  "strict": true
}
```
