ArcGIS REST API supports responses in several formats. You specify the response format using the query parameter `f` . A list of valid formats is included for each of the resources and operations where `f`  is a parameter.

### HTML

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=html` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Default format unless otherwise stated. The response is an HTML page.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

The set of these HTML pages for each resource in the system is called the Services Directory. To view the Services Directory page for the root directory of ArcGIS Enterprise, you would use the following URL:

```javascript
https://machine.domain.com/webadaptor/rest/services?f=html
```

 

Since "html" is the default value, you are not required to include this parameter in the URL. This means that the above URL is equivalent to the following URL:

```javascript
https://machine.domain.com/webadaptor/rest/services
```

 

### JSON

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=json` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is a [JSON](http://json.org/) object in Esri JSON format.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

This format is used by many Esri clients and APIs, including ArcGIS API for JavaScript. To retrieve the information in a JSON object, you would use the following URL:

```javascript
https://machine.domain.com/webadaptor/rest/services?f=json 
```

 

You can also reference a callback function in the URL as follows:

```javascript
https://machine.domain.com/webadaptor/rest/services?f=json&callback=myMethod
```

 

If you want the JSON object to be more readable, you can use `pjson` . You should not include this parameter in your production applications, as it will affect performance. Use the parameter for debugging purposes only:

```javascript
https://machine.domain.com/webadaptor/rest/services?f=pjson 
```

 

### GeoJSON

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=geojson` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is in [GeoJSON](http://geojson.org/) format. The GeoJSON format is standardized in [RFC 7946](https://tools.ietf.org/html/rfc7946).

</StyledTableCell>

</StyledTableRow>

</StyledTable>

<Note type={"note"}>

REST services with a `hasM`  value of `true`  do not support GeoJSON as an output format. Attempting to use the GeoJSON format with these services will result in 404 errors.

</Note>

This format is used by many non-Esri APIs and applications. Supported by query operations in ArcGIS Enterprise:

```javascript
https://machine.domain.com/webadaptor/rest/services/Census/MapServer/3/query?where=state_name='california'&f=geojson
```

 

### PBF

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=pbf` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is in the Protocol Buffers format.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

The PBF format is binary and is thus smaller and faster for clients to parse than JSON. It is used by newer versions of some Esri APIs and applications to improve efficiency. The PBF format is supported by some feature service operations from ArcGIS Enterprise 10.7 and later.

### Image

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=image` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is a streamed image. No other information is included in the response.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

```javascript
https://machine.domain.com/webadaptor/rest/services/Specialty/ESRI_StateCityHighway_USA/MapServer/export?bbox=-117,35.79,-122,42.38&f=image
```

 

### KMZ

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=kmz` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is a KML document wrapped in a `.kmz`  file.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

It can be a footprint or the result of an operation. In the following example, a request is made for a KML footprint of a map service with an extent of the United States:

```javascript
https://sampleserver1.arcgisonline.com/ArcGIS/rest/services/Specialty/ESRI_StateCityHighway_USA/MapServer?f=kmz
```

 

For more information about KML as an output option, see KML support.

### Help

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=help` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is a context-sensitive help document.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

The following URL opens the help on the page that provides information about the map service resource:

```javascript
https://machine.domain.com/webadaptor/rest/services/World_Physical_Map/MapServer?f=help
```

 

### Pitemx

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=pitemx` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response generates `.pitemx`  files for map services. These files can be opened in ArcGIS Pro 2.3 and later. 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

The `.pitemx`  file format is supported by ArcGIS Enterprise 10.7 and later.

### Lyr

<Note type={"note"}>

With the release of ArcGIS Enterprise 11.0, support for ArcMap has been deprecated.

</Note>

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=lyr` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Generates a layer file for viewing in ArcMap.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

```javascript
https://machine.domain.com/webadaptor/rest/services/World_Physical_Map/MapServer?f=lyr 
```

 

### Jsapi

<StyledTable headers={"Parameter format,Description"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f=jsapi` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response is a web page for viewing a map service in a web browser using ArcGIS API for JavaScript.

</StyledTableCell>

</StyledTableRow>

</StyledTable>

```javascript
https://machine.domain.com/webadaptor/rest/services/World_Physical_Map/MapServer?f=jsapi 
```
