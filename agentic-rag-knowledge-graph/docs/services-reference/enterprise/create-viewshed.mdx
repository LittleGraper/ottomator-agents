<StyledTableCell cellStyle="markdown">

`optimizeFor` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Specifies the optimization method that will be used to calculate the viewshed.

The options are as follows:

-   `SPEED`—Processing speed will be optimized, trading some accuracy in the result for higher performance. This is the default.
-   `ACCURACY`—Accuracy will be optimized, at the expense of a longer processing time.

Syntax: A string describing the optimization method.

Example:

```javascript
"optimizeFor": "ACCURACY"
```

 

</StyledTableCell>

</StyledTableRow>

<StyledTableCell cellStyle="markdown">

`maximumViewingDistance` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The cutoff distance where the computation of visible areas stops. Beyond this distance, it is unknown whether the analysis points and the other objects can see each other.

The value must be supplied through either the `maximumViewingDistance`  parameter or the `maximumViewingDistanceField`  parameter.

<GUID_512E1710_163F_43F9_8ABC_FA35AE6DD5DF_GUID_2F2E787E_6290_45AC_8EAF_5FCDBF419470 />

<GUID_512E1710_163F_43F9_8ABC_FA35AE6DD5DF_GUID_F768CD81_4907_45AF_8B6C_E9A14160A796 />

Example:

```javascript
"maximumViewingDistance": {"distance": <value>, "units": <unit>}
```

 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`maximumViewingDistanceField` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

A field that specifies the maximum viewing distance for each observer. You can use any numerical field from the input observer point features.

The value contained in this field must be in the same unit as the x,y unit of the input elevation surface.

<GUID_512E1710_163F_43F9_8ABC_FA35AE6DD5DF_GUID_AE7CF00F_779F_4482_912C_E9352F448790 />
