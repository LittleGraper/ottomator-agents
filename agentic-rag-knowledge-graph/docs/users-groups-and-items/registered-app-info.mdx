## Description

If the parent item is registered using the [register app](/register-app) operation, this resource returns information pertaining to the registered app. Every registered app gets an App ID and App Secret which in OAuth speak are known as `client_id` and `client_secret` respectively.

Note that this resource (`registeredAppInfo` resource) returns the same response as the [registered app](/registered-app) resource. The difference is that since this resource is available as a child of the [user item](/item) resource you can access this resource knowing just the item's `itemId`. Whereas accessing the registered app resource requires knowledge of the registered app's `client_id`.
