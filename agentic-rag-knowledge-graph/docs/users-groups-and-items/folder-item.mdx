## Example usage

The following is a sample ArcGIS Online request URL used to access the user item resource within a folder:

```markdown
https://org.arcgis.com/sharing/rest/content/users/jsmith/0bffc1c3fe984a17bd9ae705b3ca7e22/items/1851c185888a43fcb3d6fffb78ae7c94?f=pjson
```

The following is a sample ArcGIS Enterprise request URL used to access the user item resource within a folder:

```markdown
https://machine.domain.com/webadaptor/sharing/rest/content/users/jsmith/0bffc1c3fe984a17bd9ae705b3ca7e22/items/1851c185888a43fcb3d6fffb78ae7c94?f=pjson
```

## Description

An individual user-uploaded item link resource, within a folder, returns a list of groups with which the item is shared. The item is not physically stored in the user's folder but is stored as a link to the original item in the [item resource](/item) (`/content/items/[itemId]`). Access to the item via this link is available only to the user.

## JSON Response syntax

```other
{
  "item": {
    "id": "<item id>",
    "owner": "<owner username>",
    "created": date created shown in UNIX time,
    "modified": date modified shown in UNIX time,
    "guid": <unique id>,
    "name": "<item name>",
    "title": "<item title>",
    "type": "<type>",
    "typeKeywords": [
      "<keyword1>",
      "<keyword2>",
      "<keyword3>",
      "<keyword4>"
    ],
    "description": "<description>",
    "tags": [
      "<tag1>",
      "<tag2>"
    ],
    "snippet": "<summary>",
    "thumbnail": <file name>,

    "licenseInfo": "<access and use constraints>",
    "culture": <culture code>,
    "url": <url>
    "access": "private | public | org | shared",
    "size": <size>,
    "contentStatus": "<contentStatus>",
    "numComments": <number of comments>,
    "numRatings": <number of ratings>,
    "avgRating": <average rating>,
    "numViews": <number of views>,
    "itemControl": "admin | update",
    "scoreCompleteness": <score>,
    "groupDesignations": <special groups list>,

  }
``` 
