## Description

The `reports` operation generates reports, in CSV format, on the overall usage of the organizations. Reports define organization usage metrics in one place for a set time period (either a day, week, or month). Administrators can generate reports that monitor which organization member is using certain services, as well as how many credits and storage are used within a certain time period. Reports also include the current state of the organization, which includes the number of items, groups, users, user types, app license assignments, and public items.

## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `[Common Parameters]` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      For a complete listing, see [Common parameters](/common-parameters).

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `reportType` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The report type. Currently, only the organization (`org`) report type is supported.

      Values: `org`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `reportSubType` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Specifies the report subtype.

      Values: `content` \| `users` \| `activity` \| `credits` \| `serviceUsages` \| `itemUsages`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `timeAggregate` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      This displays the time usage aggregated by a specified value. This applies only when the `reportSubType` is set to `itemUsages`.

      Values: `day` \| `week` \| `month`

      <Note type={"note"}>

      The `timeAggregate` must be one level lower than the `timeDuration`. For example, if the `timeDuration` was set to `monthly`, then the `timeAggregate` would be `weekly` or `daily`.

      </Note>

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `timeDuration` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Specifies the time duration for the reports. 
      
      This parameter is required when `reportSubType` is set to `credits`, `activity`, `serviceUsages`, or `itemUsages`. 
      
      This parameter does not apply when the `reportSubType` is set to `content` or `users`.

      <Note type={"note"}>

      The `daily` value is only available when `reportSubType` is set to `activity`.

      The `yearly` value is only available when `reportSubType` is set to `itemUsages`.

      </Note>

      Values: `daily` \| `weekly` \| `monthly` \| `quarterly` \| `yearly`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `startTime` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The start time of the time duration. The time format is Unix time with millisecond precision. 
      
      If `timeDuration` is `weekly`, the `startTime` value must be a time on Sunday or Monday GMT. 
      
      If `timeDuration` is `monthly`, the `startTime` value must be on the first day of the month.

      This parameter does not apply when the `reportSubType` is set to `content` or `users`.

      Example:

      ```markdown
      startTime=1642982400
      ``` 

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## JSON Response syntax

The following syntax example demonstrates a response for item and member reports:

```other
{
  "itemId": "<item id>",
  "status": "processing",
  "orgId": "<org id>",
  "owner": "<owner>",
  "type": "Administrative Report",
  "created": <time in milliseconds>,
  "properties": {
    "reportType": "org",
    "reportSubType": content | users
  }
}
```

The following syntax example demonstrates a response for credit reports:

```other
{
  "itemId": "<item id>",
  "status": "processing",
  "orgId": "<org id>",
  "owner": "<owner>",
  "type": "Administrative Report",
  "created": <time in milliseconds>,
  "properties": {
    "timeDuration": daily | weekly | monthly | quarterly | yearly,
    "startTime": <startTime>,
    "reportType": "org",
    "reportSubType": content | users | activity | credits | serviceUsages | itemUsages
  }
}
```
