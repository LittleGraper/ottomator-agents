## Description

You can use this operation to change the configuration parameters and shared key.

<Note type={"caution"}>

If the shared key parameter has been changed, you need to re-register each of your web adaptors so that they can pick up the updated key. The configuration must always have the shared key parameter.

</Note>

##  Request parameters

<StyledTable headers={"Parameter,Details"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`webAdaptorsConfig` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The `sharedKey`  attribute must always be present in this JSON.

Example

```javascript
webAdaptorsConfig={"sharedKey": "XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXX"}
```

 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response format. The default value is `html` .

Values: `html`  \| `json` 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

## Example usage

Below is a sample POST request for `update` :

```javascript
POST /webadaptor/admin/system/webadaptors/config/update HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

webAdaptorsConfig={"sharedKey": "XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXX"}&f=json
```

 

## JSON Response example

```javascript
{"status": "success"}
```
