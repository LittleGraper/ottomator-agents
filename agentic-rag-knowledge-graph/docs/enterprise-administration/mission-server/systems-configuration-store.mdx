## Description

The configuration store maintains all of server's configurations. Typical configurations include all the resources such as clusters, machines, GIS services, and security rules that are required to power a [Site](/admin-site-root). In a way, the configuration store is a physical representation of a site. 

Every server machine, when it joins the site, is provided with a connection to the configuration store and it can thereafter participate in the management of the site. You can change the store's properties during runtime using the [edit](/systems-configuration-store-edit-config-store) operation. 

The Administrator API that runs on every server machine is capable of reading and writing to the store. As a result, the store must be accessible to every server machine within the site. The default implementation is built on top of a file system and stores all the configurations in a hierarchy of folders and files.

## Example usage

Below is a sample request URL for the `configstore`  resource:

```javascript
https://machine.domain.com/webadaptor/admin/system/configstore?f=json
```

 

## JSON Response syntax

```javascript
{
  "type": "FILESYSTEM", //only supported value for this property
  "connectionString": "<connection_string>",
  "class": "<class>",
  "localRepositoryPath": "<localpathOnmachine>",
  "status": "<Initializing|Ready>"
}
```

 

## JSON Response example

```javascript
{
  "type": "FILESYSTEM",
  "connectionString": "C:\\arcgisserver\\config-store",
  "class": "com.esri.arcgis.discovery.admin.store.file.FSConfigStore", //default class name for FILESYSTEM type
  "localRepositoryPath": "C:\\arcgisserver\\local",	
  "status": "Ready"
}
```
