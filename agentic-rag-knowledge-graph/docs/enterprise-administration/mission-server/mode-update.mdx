## Description

The `update`  operation is used to move between the two types of site [modes](/mode). Switching to `READ_ONLY`  mode will restart all your services as the default behavior. Moving to `EDITABLE`  mode will not restart services.

<Note type={"note"}>

The default options when enabling `READ_ONLY`  mode can be changed by configuring them in the [serverProperties](/systems-properties) resource. You can choose not to copy configurations locally and not to restart services. See the [serverProperties](/systems-properties) resource for the required format for changing these options.

</Note>

##  Request parameters

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

(Required)

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Values: `READ_ONLY`  \| `EDITABLE` 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Values: `html`  \| `json`  \| `pjson` 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

## Example usage

Below is a sample request URL for the `root`  resource:

```javascript
POST /webadaptor/admin/mode/update HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

siteMode=READ_ONLY&f=json
      
```
