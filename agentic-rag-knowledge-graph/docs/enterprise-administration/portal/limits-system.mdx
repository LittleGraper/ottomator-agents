## Description

The `limits` resource provides limits associated with the organization (such as scheduled tasks for ArcGIS Notebooks, Workflow Manager, ArcGIS Insights, or geoprocessing services) for individual users and the for an organization as a whole. 

##  Request parameters 

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Response properties

<StyledTable headers={"Property,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `UpdateInsightsUserLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of scheduled Insight reports that can be created by an individual user.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `UpdateInsightsOrgLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum total number of scheduled Insight reports for an organization.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `ExecuteNotebooksUserLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of active scheduled notebook tasks that can be owned by an individual user. By default, a user can only own 20 active notebooks tasks. When this limit is reached, the user will not be able to create a new notebook task until one of the notebook tasks change its status from `Active` to `Complete`, `Failed`, or `Inactive`. The number of ownable active tasks can be updated using the [Update Limits](/update-system-limits) operation.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `ExecuteNotebooksOrgLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of total active scheduled notebook tasks for an organization. By default, an organization is limited to 200 active notebook tasks. When this limit is reached, users will not be able to create any additional notebook tasks. The number of total active tasks for an organization can be updated using the [Update Limits](/update-system-limits) operation.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `ExecuteWorkflowManagerUserLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of scheduled recurring workflow manager jobs that can be created by an individual user.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `ExecuteWorkflowManagerOrgLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum total number of scheduled recurring workflow manager jobs for an organization.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `FailedRunsToDisableTask` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The number of times a task can fail consecutively before it is switched to a failed state. By default, a task can fail 5 times. When this limit is met, the task is changed from its active state, and the task owner must identify and rectify the issue before switching the task's access back to Active. The failure limit can be updated using the [Update Limits](/update-system-limits) operation.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `TaskRunHistoryCount` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The total number of runs that are reported and maintained for a scheduled task. By default, only the 30 most recent runs will be maintained. Any tasks that are not within the most recent 30 runs will be permanetly deleted. The number of runs can be increased to a total of 50 runs per scheduled task using the [Update Limits](/update-system-limits) operation.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `GPServiceUserLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of active scheduled geoprocessing service tasks that can be owned by an individual user.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `GPServiceOrgLimit` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum total number of active scheduled geoprocessing service tasks for an organization.

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Example usage

The following is a sample request URL used to access the `limits` resource:

```markdown
https://machine.domain.com/webadaptor/portaladmin/system/limits?f=json
```
