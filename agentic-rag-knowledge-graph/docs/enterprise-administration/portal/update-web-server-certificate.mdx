## Description

The `update` operation configures the web server certificate, TLS protocols, and cipher suites used by the portal.

<Note type={"note"}>

This operation is not accessible for highly available portals. In such instances, a counterpart [Update Web Server Certificate](/update-web-server-certificate-ha) operation should be used in its place for a high availability workflow. For more information on high availability, please see [Configure a highly available portal](https://enterprise.arcgis.com/en/portal/latest/administer/windows/configuring-a-highly-available-portal.htm).

</Note>

## Example usage

The following is a sample POST request for the `update` operation:

```other
POST /webadaptor/portaladmin/security/sslCertificates/update HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

webServerCertificateAlias=myselfsignedcertificate&sslProtocols=TLSv1.2&HSTSEnabled=false&cipherSuites=TLS_RSA_WITH_AES_256_CBC_SHA&f=json

```
