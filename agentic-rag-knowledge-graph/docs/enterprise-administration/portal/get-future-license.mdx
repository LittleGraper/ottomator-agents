## Description

The `getFutureLicense` operation returns your future user types and add-on license information. If your license includes future entitlements, this operation will return the maximum number of registered members per license and the date (Unix format) that it will go into effect. If your license does not have any future entitlements, this operation will still return the same future licensing information; however, the number of maximum registered members will be listed as 0 and the `startDate` response parameter will instead return the license's expiration date.

## Example usage

The following is a sample GET request for the `getFutureLicense` operation:

```markdown
https://machine.domain.com/webadaptor/license/getFutureLicense?f=pjson
```

## JSON Response syntax

```other
{
  "userTypes": [
    {
      "id": <"user type ID">,
      "maximumRegisteredMembers": <number of maximum registered members>,
      "startDate": <date in epoch time>
    }
  ],
  "appBundles" [
    {
      "id": <"app bundle ID">,
      "itemId": <"item ID">,
      "maximumRegisteredMembers": <number of maximum registered members>,
      "startDate": <date in epoch time>
    }
  ],
  "apps": [
    {
      "id": <"app ID">,
      "itemId": <"item ID">,
      "maximumRegisteredMembers": <number of maximum registered members>,
      "startDate": <date in epoch time>
    }
  ],
  "extensions": [  //New at 10.8
    {
      "id": "<extension ID>",
      "itemId": "<item ID>",
      "maximumRegisteredMembers": <number of maximum registered members>,
      "startDate": <date in epoch time>
    }
  ],
  "portalCapabilities": []
}
```
