## Description

The `updateEnterpriseUser` operation allows an administrator to update the `idpUsername` for an organization member with an enterprise account. This is used when migrating from accounts used with web-tier authentication to SAML authentication.

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `username` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Username of the enterprise account.

      ```javascript label=Markdown
      username=john
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `idpUsername` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Username used by the SAML identity provider.

      ```markdown label=Example
      idpUsername=john@company.com
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default value is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Example usage

The following is a sample POST request for the `updateEnterpriseUser` operation:

```markdown
POST /webadaptor/portaladmin/security/users/updateEnterpriseUser HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

username=john&idpUsername=john@company.com&f=pjson
```
