## Description

The `getEnterpriseGroupsForUser` operation lists the groups assigned to a user account in the configured enterprise group store. You can use the `filter` parameter to narrow down the search results.

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `username` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The name of the user account for which to return the assigned groups.

      ```markdown label=Example
      username=domain\jdoe
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `filter` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      An optional wildcard string that can be used to filter a search result to return specific groups.

      ```other label=Example
      filter=*bldg*
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `maxCount` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of results the client is willing to accept.

      ```markdwon label=Example
      maxCount=10
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `getEnterpriseGroupsForUser` operation:

```other
POST /webadaptor/portaladmin/security/groups/getEnterpriseGroupsForUsers HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

username=domain\jdoe&filter=*bldg*&maxCount=10&f=json
```
