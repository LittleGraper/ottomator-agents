## Description

The `edit` operation updates the log settings for the portal.

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `logDir` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      File path to the root of the log directories. The default for Windows is `C:\arcgisportal\logs` and for Linux is `[server root]/usr/logs/`.

      Example

      ```markdown label=Example
      logDir=C:\arcgisportal\logs
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `logLevel` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The highest level of detail at which logs will be recording. The default is `WARNING`.

      Values: `OFF` | `SEVERE` | `WARNING` | `INFO` | `FINE` | `VERBOSE` | `DEBUG`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `maxLogFileAge` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      By default, logs are retained on disk for a period of approximately 90 days. When a log exceeds this time frame, it is deleted. Specifying a smaller time frame, for example, `30` days, can prevent the log directory from growing larger than expected. The default is `90` (days).

      ```markdown label=Example
      maxLogFileAge=90
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format: The default is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `edit` operation:

```markdown
POST /webadaptor/portaladmin/logs/settings/edit HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

logDir=C:\arcgisportal\logs&logLevel=WARNING&maxLogFileAge=90&f=json
```
