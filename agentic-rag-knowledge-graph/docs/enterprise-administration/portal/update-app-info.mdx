## Description

The `updateAppInfo` operation allows administrators to update the redirect URIs for a registered application. Use the [Get App Info](/get-app-info) operation to obtain the existing URI values for the application.

##  Request parameters 

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `appInfo` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The JSON object representing the applications information.

      ```markdown label=Example
      appInfo={"appId": "mapsForDynamicsCRM", "redirectURIs": ["https://machine.domain.com","https://crm.domain.com/maps"]}
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `updateAppInfo` operaiton:

```markdown
POST /webadaptor/portaladmin/security/oauth/updateAppInfo HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

appInfo={
  "appId": "mapsForDynamicsCRM",
  "redirectURIs": [
    "https://machine.domain.com",
    "https://crm.domain.com/maps"
  ]
}&f=pjson
```

## JSON Response example

```markdown
{"status": "success"}
```
