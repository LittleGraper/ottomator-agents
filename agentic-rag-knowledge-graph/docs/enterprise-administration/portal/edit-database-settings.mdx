## Description

The `edit` operation can be used to disable incremental backups for an organization if they have been enabled using the webgisdr utility. This operation cannot be used to manually enable incremental backups. For information on how to enable incremental backups, see [Create an ArcGIS Enterprise backup](https://enterprise.arcgis.com/en/portal/latest/administer/windows/create-web-gis-backup.htm) and [Configure disaster recovery for ArcGIS Enterprise](https://enterprise.arcgis.com/en/portal/latest/administer/windows/configure-disaster-recovery.htm).

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `incrementalBackupEnabled` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Controls whether the portal will create transaction logs, which are used when creating incremental backups. The only value the `incrementalBackupEnabled` accepts is `false`.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f`

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Example usage

The following is a sample POST request for the `edit` operation:

```markdown
POST /webadaptor/portaladmin/system/database/settings/edit HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

incrementalBackupEnabled=false&f=pjson

```
