## Description

The `test` operation sends a test email via the SMTP server to ensure that the organization's email settings have been properly configured. If the settings were configured successfully, an email will be sent out to a specified email address (`mailTo`).

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `mailTo` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The email the test message will be sent to. The default value for this parameter will be the email address that has been assigned to `fromEmailAddress`, set through the [Update](/update-email-settings) operation.

      ```javascript label=Example
      mailTo=admin@myorg.com
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Example usage

The following is a sample POST request for the `test` operation:

```markdown
POST /webadaptor/portaladmin/system/emailSettings/test HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

mailTo=admin@myorg.com&f=pjson

```
