## Description

ArcGIS Notebook Server has a site mode that allows you to control changes to your site. You can set the site mode to `READ_ONLY`  to prevent the opening of new notebooks and block most administrative operations. The default site mode is `EDITABLE` , which allows all administrative and notebook operations on the site.

## Response properties

<StyledTable headers={"Property,Details"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`siteMode` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The current mode of the site. 

Response: `READ_ONLY`  \| `EDITABLE` .

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`copyConfigLocal` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Indicates whether the site configuration files will be copied to the local repository upon switching to `READ_ONLY` . 

Response: `true`  \| `false` 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`lastModified` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Time stamp that indicates the last time the site mode was modified. 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

## JSON Response syntax

```javascript
{
    "siteMode": READ_ONLY or EDITABLE,
    "copyConfigLocal": true or false,
    "lastModified": <timestamp>
}
```

 

## JSON Response example

```javascript
{
    "siteMode": "EDITABLE",
    "copyConfigLocal": false,
    "lastModified": <timestamp>
}
```
