## Description

This operation lists files that are located in the workspace directory (`/arcgis/home` ) of the user making the request. A list of all files and folders is returned in a single response. Files contained in a sub folder are returned in a `foldername/filename`  format.

<Note type>

The `Transfer User Workspace` resource can also be accessed by the `/dataaccess/notebookworkspace` endpoint. 

</Note>

##  Request parameters

<StyledTable headers={"Parameter,Details"}>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`restype` 

(Required)

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The default container is the workspace directory of the user.

Value: `container` 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`comp` 

(Required)

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Specifies the operation to list all files.

Value: `list` 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`f` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The response format. The default response format is xml.

Values: `html`  \| `json`  \| `pjson`  \| `xml` 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

## Example usage

Below is a sample request URL for the list files operation:

```javascript
https://notebookserver.domain.com/arcgis/admin/dataaccess/notebookworkspace?restype=container&comp=list&f=html
```
