## Description

The `start` operation starts the server machine if it's participating in a site. As it starts, it pulls all the GIS [service](/service) configurations that have been deployed to the site and prepares to host them. This operation can take awhile to complete. The [status](/status) resource indicates the current status of the server machine.

## Example usage

The following is a sample POST request for the `start` operation:

```markdown
POST /webadaptor/admin/machines/MACHINE.DOMAIN.COM/start HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

f=pjson
```
