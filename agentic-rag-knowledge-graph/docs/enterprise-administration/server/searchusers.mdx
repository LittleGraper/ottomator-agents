## Description

The `search` operation searches for a specific user, or a group of users, from the [user store](/userstore). The size of the search result can be controlled with the `maxCount` parameter.


##  Request parameters

<StyledTable headers={"Parameter,Description"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `filter` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A filter string to search the users.

      ```markdown label=Example
      filter=john
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `maxCount` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of users that will be returned.

      ```markdown label=Example
      maxCount=20
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`. 

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `search` operation that seeks to return the first 20 users whose usernames start with 'john':

```markdown
POST /webadaptor/admin/security/users/search HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

filter=john&maxCount=20&f=json
```

## JSON Response syntax

```other
{
  "users": [<user1>, <user2>],
  "hasMore": <true|false>
}
```

## JSON Response example
