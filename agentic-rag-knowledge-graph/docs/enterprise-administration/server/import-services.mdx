## Description

The `importServices` operation imports services from a server that were exported using the [Export Services](/export-services) operation.

<Note type={"caution"}>

Importing Services can take some time. Upon completion, the operation will print a report on the import services page. The report is not persisted on the server and will be lost if you navigate away from the import services page. Some of the messages and warnings generated by this operation will be accessible in the logs.

</Note>


##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `location` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The upload ID of, or folder path to, the .agssiteservices file that was generated from a run of the [Export Services](/export-services) operation.

      ```markdown label=Example
      //ID example
      location=i5fad5582-80bc-4155-a21c-62f326354aba

      //File path example
      location=\\server\share\backup\<timestamp>.agssiteservices
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `importServices` operation:

```javascript
POST /arcgis/admin/services/importServices HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

location=i5fad5582-80bc-4155-a21c-62f326354aba&f=json
```
