## Description

The `remove` operation removes an existing role from the [role store](/rolestore). This is available only when the role store is a read-write store, such as the default ArcGIS Server store.


## Request parameters

<StyledTable headers={"Parameter,Description"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `rolename` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The name of the role to remove.

      ```markdown label=Example
      rolename=editors
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`. 

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `remove` operation:

```markdown
POST /webadaptor/admin/security/roles/remove HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

rolename=editors&f=pjson
```
