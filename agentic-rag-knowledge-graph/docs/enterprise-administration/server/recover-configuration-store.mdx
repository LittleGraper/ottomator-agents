## Description

The `recover` operation can be used to recover a site if the shared configuration store is permanently lost. If the shared configuration store for a site is unavailable, a site in read-only mode will operate in a degraded capacity that allows access to the ArcGIS Server Administrator Directory. The site must be in read-only mode, and the site configuration files must have been copied to the local repository when switching site modes. The `recover` operation will copy the configuration store from the local repository into the shared configuration store location. The copied local repository will be from the machine in the site where the `recover` operation is performed.

## Example usage

The following is a sample POST request for the `recover` operation:

```markdown
POST /webadaptor/admin/system/configstore/recover HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

f=pjson
```
