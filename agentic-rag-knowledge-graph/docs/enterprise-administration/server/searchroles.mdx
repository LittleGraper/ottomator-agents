## Description

The `search` operation searches for a specific role, or a group of roles, from the [role store](/rolestore). The size of the search results can be controlled with the `maxCount` parameter.


## Request parameters

<StyledTable headers={"Parameter,Description"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `filter` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A filter string to search the roles.

      ```markdown label=Example
      filter=editors
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `maxCount` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum size of the result size.

      ```markdown label=Example
      maxCount=20
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `search` operation that seeks to return roles that begin with the name 'editors':

```markdown
POST /webadaptor/admin/security/roles/search HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

filter=editors&maxCount=20&f=json
```

## JSON Response example

```javascript
{
  "roles": [editors],
  "hasMore": false
}
```
