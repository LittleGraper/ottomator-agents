## Description

The `properties` resource is a container for ArcGIS Server configuration properties that govern its behavior. The properties are available to all server objects and extensions through the server environment interface.


##  Request parameters

  <StyledTableRow>

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

## Server properties

<StyledTable headers={"Property,Details"}>
  <StyledTableRow>
  <StyledTableCell cellStyle="markdown">

      `allowGPAndExtensionPublishingToPublishers` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Allows both publishers and administrators to publish geoprocessing services and extensions if set to `true`. Only administrators can publish extensions and geoprocessing services by default.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `allowLegacyURLParser` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      When added and set to `true`, braces `{}`, brackets `[]`, pipe `|`, or double quotation marks `"` special characters are allowed in URL query strings unencoded. By default, the property is not listed and its value is `false`. It is recommended that you do not enable this property, as it carries a security vulnerability. If you do add or change the property, manually restart ArcGIS Server for it to take effect.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `CacheSizeForSecureTileRequests` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      An integer that specifies the number of users whose token information will be cached. This increases the speed of tile retrieval for cached services. If not specified, the default cache size is 200,000. Both REST and SOAP services honor this property. You'll need to manually restart ArcGIS Server for this change to take effect.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `defaultJsonResponseContentType` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Changes the content-type for responses from `application/json`, `application/javascript`, or `application/geo+json` to `text/plain`. The default content-type for JSON and JSON with callback responses are `application/json` and `application/javascript`, respectively. The default content-type for GeoJSON is `application/geo+json`. 

      All three of these content-type properties can be reverted in your ArcGIS Services and Server Administration REST APIs back to `text/plain` by setting this property to `text/plain`. Restarting ArcGIS Server will cause the change to take effect. You should carefully consider the security risks of changing the content-type of JSON and GeoJSON responses on your server.

      Values: `text/plain`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `DisableAdminDirectoryCache` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Disables browser caching of the Administrator Directory pages. The default is `false`. To disable browser caching, set this property to `true`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `disableIPLogging` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      When a possible cross-site request forgery (CSRF) attack is detected, the server logs a message containing the possible IP address of the attacker. If you do not want IP addresses listed in the logs, set this property to `true`. Also, HTTP request referrers are logged at FINE level by the REST and SOAP handles unless this property is set to `true`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `enableNosniffHeader` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      When set to `true`, all network communication from ArcGIS Server includes the `X-Content-Type-Options` response header with a value of `nosniff`. This helps protect against MIME confusion attacks by preventing web browsers from MIME sniffing a response away from the declared content-type of an asset. The default value is `true`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `featureServiceXSSFilter` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      This parameter is used to set the value of the `xssPreventionRule`vfor feature services when they are published. When set to `input`, an XSS filter is applied to any attribute data input into a feature service for new or updated features. When set to `inputOutput`, both the input data and the output data are filtered for XSS strings; this is a more secure option but may have a performance impact when applied to some feature services. Changes made to this parameter only apply to new feature services and will not update the `xssPreventionRule` for existing feature services. The default value is `input`.

      Values: `input` | `inputOutput`

    </StyledTableCell>
  </StyledTableRow>

      `ldapCertificateValidation`

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      When set to `true`, any encrypted LDAP communication (LDAPS) from ArcGIS Server to the user or role identity store will enforce certificate validation. The default value is `false`. When set to `true`, if the certificate name does not match or the certificate cannot be validated, ArcGIS Server may attempt to use unencrypted LDAP communication or it may not be able to establish communication at all. If this occurs, a warning message will be logged.

      Values: `true` | `false`

  </StyledTableRow>

    <StyledTableCell cellStyle="markdown">

      `localTempFolder` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Defines the local folder on a machine that can be used by GIS services and objects. If this property is not explicitly set, the services and objects will revert to using the system's default temporary directory. If this property is used, you must create the temporary directory on every server machine in the site.

      ```markdown label=Example
      /tmp/arcgis
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `machineSuspendThreshold` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Machines record their status once per minute in the configuration store if they are active. This property defines the time period of inactivity after which a machine will be suspended from receiving administrative and publishing requests. Once a suspended machine records an active status, it will be reinstated to receive such requests. The default value is 60 minutes. See `suspendedMachineUnregisterThreshold`.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `messageFormat` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Defines the transmission protocol supported by the services catalog in the server.

      Values: `esriServiceCatalogMessageFormatBin` | `esriServiceCatalogMessageFormatSoap` | `esriServiceCatalogMessageFormatSoapOrBin`

    <StyledTableCell cellStyle="markdown">

      `messageVersion`

      Defines the version supported by the services catalog in the server.

      ```markdown label=Example
      esriArcGISVersion107
      ```

    </StyledTableCell>

  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `minutesToCacheUsersAndRoles` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Defines the time period, in minutes, for which users and rules from your LDAP or Active Directory will be cached on the server after a request for users or roles form the identity store. This optimizes the performance of your secure services. By default, the time period is 30 minutes. You can modify the time period using this property, or you can disable caching by setting the property to zero.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `nonProxyHosts` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      This property should always contain the machine name where your portal is installed. If you want to federate ArcGIS Server with your portal, this property should also include the name of the machine where ArcGIS Server is installed. If the ArcGIS Server site contains multiple machines, a wildcard can be used for all machines in the domain (for example, `*.domain.com`). Machine and domain items are separated using a pipe `|`. 

    </StyledTableCell>
  </StyledTableRow>

    <StyledTableCell cellStyle="markdown">

  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `percentageMaxAllowedComputeMemory` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Sets the maximum percentage of memory, out of the total amount of memory available on the machine, that will be used by the compute platform worker process on a single node.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `PushIdentityToDatabase` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Propagates the credentials of the logged-in user to make connections to an Oracle database. This property is only supported for use with Oracle databases.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `readOnlyModeCopyConfigLocal` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Determines whether configuration files are copied to the local repository when enabling read-only mode. The default value is `true`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `readOnlyModeRestartServices` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Determines the service restart behavior when enabling read-only mode. The default is `true`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `standardizedQueries` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      This security property forces developers to use [standardized SQL queries](https://enterprise.arcgis.com/en/server/latest/administer/windows/about-standardized-queries.htm) in where clauses when working with map, feature, image, and WFS services through REST or SOAP. This makes it easier for developers and applications to query ArcGIS Server services and helps prevent SQL injection attacks. Standardized queries are enforced by default but can be disabled by the server administrator.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
      <StyledTableCell cellStyle="markdown">

      `suspendDuration` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Specifies the duration the ArcGIS service hosting processes should suspend at startup. This duration is specified in milliseconds. This is an optional property that takes effect when `suspendServiceAtStartup` is set to `true`. If the duration is not specified and suspension of service at startup is requested, the default suspend duration is 30 seconds.

      ```markdown label=Example
      10000 //duration set for 10 seconds
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `suspendedMachineUnregisterThreshold` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Machines record their status once per minute in the configuration store if they are active. This property defines the time period of inactivity after which a machine will be unregistered from the site. This property is disabled by default, with a value of `-1`. If used, this threshold should be higher than the `machineSuspendThreshold` property.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `suspendServiceAtStartup` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Suspends the ArcGIS service hosting processes at startup. This will enable attaching to those processes and debugging code that run early in the life cycle of server extensions soon after they are instantiated.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `uploadFileExtensionAllowedList` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"note"}>

      At 10.9, the name of this property has been updated from `uploadFileExtensionWhitelist` to `uploadFileExtensionAllowedList`.

      </Note>

      Specifies which files are allowed to be uploaded through the file upload API by identifying the allowable extensions. It is a list of comma-separated extensions without dots. If this property is not specified, a default list is used.

      <Note type={"note"}>

      Updating the list overrides the default list completely. This means that if you set this property to a subset of the default list, only those items in the subset will be accepted for upload.

      </Note>

      Default list: `soe` | `sd` | `sde` | `odc` | `csv` | `txt` | `zshp` | `kmz` | `geodatabase`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `uploadItemInfoFileExtensionAllowedList` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"note"}>

      At 10.9, the name of this property has been updated from `uploadItemInfoFileExtensionWhitelist` to `uploadItemInfoFileExtensionAllowedList`.

      </Note>

      Specifies which files are allowed to be uploaded through the service `iteminfo` upload API by identifying the allowable extensions. It is a list of comma-separated extensions without dots. If this property is not specified, a default list is used.

      <Note type={"note"}>

      This list overrides the default list completely. This means that if you set this property to a subset of the default list, only those items in the subset will be accepted for upload. Example: `png`, `svg`, `gif`, `jpg`, `tiff`, `bmp`.

      </Note>

      Default list: `xml` | `img` | `png` | `gif` | `jpg` | `jpeg` | `bmp`

    </StyledTableCell>
  </StyledTableRow>

  <StyledTableRow>

      `zkDebug` 

    </StyledTableCell>

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

    </StyledTableCell>
  </StyledTableRow>

## Example usage

The following is a sample request URL used to access the `properties` resource:

```markdown
https://machine.domain.com/webadaptor/admin/system/properties?f=pjson
```

## JSON Response syntax

```javascript
{
  "<property1>": <value1>,
  "<property2>": <value1>,
  ...
}
```
