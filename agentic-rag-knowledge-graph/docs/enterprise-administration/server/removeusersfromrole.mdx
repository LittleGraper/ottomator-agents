## Description

The `removeUsersFromRole` operation removes a role assignment from one or multiple users.

##  Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `rolename` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The name of the role to be removed.

      ```markdown label=Example
      rolename=editors
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `users` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A comma-separated list of usernames that have been assigned the designated role. Each username must already exist in the user store.

      ```markdown label=Example
      users=njones,aturing
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `removeUsersFromRole` operation:

```markdown
POST /webadaptor/admin/security/roles/removeUsersFromRole HTTP/1.1
Host: machine.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

rolename=editors&users=njones,aturing&f=pjson
```
