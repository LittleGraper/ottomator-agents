## Description

<Note type={"note"}>

The request parameters, syntax, and examples described in this topic can be used in Insights 3.0 and lower. The parameters and syntax are deprecated starting at Insights 3.1.

</Note>

Relational Data Store Types are used by ArcGIS Insights. They enable Insights users to browse and query data from relational databases through the creation of Relational Database Connection portal items.

<Note type={"note"}>

Relational data store types are only used with Insights and are not to be confused with ArcGIS Data Store relational data stores.

</Note>

This resource lists the relational data store types that have been [registered](/register-relational-data-store-type) with the server. Each registered relational data store type has both an `id` and a `name` property, as well as an array of `userDefinedProperties`, which indicates the properties that client applications must provide when creating a Relational Database Connection [portal item](DEVSITE/rest/users-groups-and-items/items-and-item-types). Only administrators can [register](/register-relational-data-store-type) and [unregister](/unregister-relational-data-store-type) a relational data store type. 

<Note type={"note"}>

ArcGIS Insights must be installed in order to register Relational Data Store Types and create Relational Catalog Services or Relational Database Connections. Please refer to ArcGIS Insights documentation for more information on [supported databases](https://doc.arcgis.com/en/insights/2023.2/administer/supported-databases.htm), [required vendor files](https://doc.arcgis.com/en/insights/2023.2/administer/vendor-files.htm), [supported data types](https://doc.arcgis.com/en/insights/2023.2/get-started/supported-types-from-databases.htm), and [required database privileges](https://doc.arcgis.com/en/insights/2023.2/administer/database-privileges.htm).

</Note>

## Example usage

The following is a sample request URL for the `relationalDataStoreTypes` resource:

```markdown
https://machine.domain.com/webadaptor/admin/data/relationalDatastoreTypes?f=pjson
```

## JSON Response syntax

```other
{
  "relationalDatastoreTypes": [  
    {  
      "name": "",
      "id": "",
      "userDefinedProperties": [] //platform-specific connection properties  
    }
  ]
}
```
