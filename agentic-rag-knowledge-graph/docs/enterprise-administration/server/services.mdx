## Description

The `services` resource acts as the root folder and container for all subfolders and GIS services. You can create a new subfolder by using the [Create Folder](/createfolder) operation and a new GIS service by using the [Create Service](/createservice) operation.

## Request parameters

<StyledTable headers={"Parameter,Description"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `detail` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Indicates that a detailed description is expected. The default value is `false`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample request URL used to access the `services` resource:

```markdown
https://machine.domain.com/webadaptor/admin/services?detail=true&f=pjson
```

## JSON Response syntax

```other
{
  "folderName": "/",
  "description": "Root folder",
  "webEncrypted": <webEncrypted>,
  "isDefault": <isDefault>,
  "folders": [
    "<folderName1>",
    "<folderName2>"
  ],
  "foldersDetail": [
    {
      "folderName": "<folderName>",
      "description": "<description>",
      "webEncrypted": <webEncrypted>,
      "isDefault": <isDefault>
    }
  ],
  "services": [
    {
      "folderName": "<folderName>",
      "serviceName": "<serviceName>",
      "type": "<serviceType>",
      "description": "<description>"
    }
  ]
}
```

The full name of a service is represented in JSON as follows

```javascript
{
  "folderName”: <folderName>,
  "serviceName”: <name>,
  "serviceType”: <type>,
  "description”: <description>
}
```
