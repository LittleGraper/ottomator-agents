## Description

The Token Manager is a web service that issues tokens. Tokens are used instead of user credentials when consuming GIS services that have token-based security enabled on them. Tokens are also used for [ consuming the Administrator API](/apisecurity) through a script or application. Token Manager exposes a REST-ful interface along with an HTML page to fetch tokens manually. An instance of the token manager is integrated into the Administrator API and can be reached at:

```javascript
https://server:6080/arcgis/admin/generateToken
```

 

For backwards compatibility, another instance of token manager is available at:

```javascript
https://server:6080/arcgis/tokens/generateToken
```

 

However, the above instance of Token Manager is configured by the administrator and started up only when token-based [ security](/updatesecurityconfig) is enabled on the GIS server. Use the [ update token configuration](/updatetokenconfig) operation to change the token manager.

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

Example

```javascript
1440
```

 

</StyledTableCell>

</StyledTableRow>

<StyledTableRow>

<StyledTableCell cellStyle="markdown">

`shortTimeout` 

</StyledTableCell>

<StyledTableCell cellStyle="markdown">

The shortest amount of time in minutes for which a token is issued.

Example

```javascript
60
```

 

</StyledTableCell>

</StyledTableRow>

</StyledTable>

## JSON Response syntax

```javascript
{
		"type": "BUILTIN",
		"properties": <properties>
}
	
```

 

## JSON Response example

The JSON representation of a Token Manager configuration: 

```javascript
{
	 "type": "BUILTIN",
	 "properties": {
		 "sharedKey": "my really really secret passphrase",
		 "longTimeout": "2880",
		 "shortTimeout": "120",
	 }
}
```
