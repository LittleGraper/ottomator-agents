## Description

The `register` operation directs the server to reserve space for a new item to be uploaded that is made up of one or more parts. Once the operation is completed, you must use the [Upload Part](/upload-part) operation to upload individual parts. After all the parts are uploaded, you must use the [Commit](/commit-upload) operation to consolidate the parts.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `itemName` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The item's file name and extension.

      ```other label=Example
      itemName=TestUpload_WaterDistributionNetwork.kmz
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `description` 

      (Optional)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The item's description.

      ```markdown label=Example
      description=Test upload of Water Distribution Network template
      ```

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>
