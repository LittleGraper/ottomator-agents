## Description

The `createFolder` operation creates a subfolder under the root Services folder. The `createFolder` operation is only supported on the root folder, as ArcGIS Enterprise on Kubernetes only supports a single-level hierarchy of folders.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `folderName` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The name of the folder. The name must be unique.

      <Note type={"note"}>

      The folder name must not contain any spaces. Attempting to pass in a folder name with spacing will result in the request failing, returning an error response.

      </Note>

      ```markdown label=Example
      folderName=SalzburgCity
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `createFolder` operation:

```other
POST /context/admin/services/createFolder HTTP /1.1
Host: organization.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

folderName=SalzburgCity&f=psjon&token=ELIMsyURgefHIZ-x6PzynYsaxLNR30GbWE6PCpgD0Hu7dWQ3vmBLz3bljwvJC8a74K_XARVfs5lpmCkjLkMZANHIDgMHiwgwnNiQBbTWHEbNqJeodE9ipXYeUmlGkveMQ44JjNQlNGYim-LqV4DcrPynxBlvO0CsIT4RIB-OsYp6oU_jqS9xVHxcHwCtvu3rnikAqeb3S96aS0ntC-414Y23t1CWAVhDSmRrl7NyWgw-jN4VBZ3FvPnDDDoFooY

```

## JSON Response example

The following sample response is returned when `createFolder` is successful:

```other
{"status": "success"}
```

The following sample response is returned when `folderName` contains spacing:

```other
{
  "error" : {
    "code" : 500,
    "message" : "Folder name is invalid. The name contains illegal character ' '.",
    "details" : null
  }
}
```
