## Description

The `upgrade` operation upgrades, through either a patch or a release, an ArcGIS Enterprise on Kubernetes deployment to the current version.

Before performing an upgrade, the unique ID associated with the patch or release must be retrieved from the version manifest using the [Available](/check-available-updates) operation. The version manifest is a JSON array of version objects that contain update-specific information, including a JSON array of container objects that specify affected containers and include their name, checksum, and image values.

Once the ID has been retrieved, you must also retrieve the [required upgrade settings](/get-upgrade-settings) that will be passed through as part of the `upgrade` operation. Some settings will require user input before they can be used during an upgrade. For more information about current upgrade settings, see the [Upgrade settings section below](/upgrade#upgrade-settings).

<Note type={"note"}>

The organization is put into [read-only mode](/mode) during an upgrade.

</Note>

Once the `upgrade` job request has been submitted, the deployment will either install a new patch on the base version or upgrade the entire deployment to the latest release. While the job is running, the [Upgrades](/upgrades) resource will return detailed, real-time job messages and status information. The `upgrades` resource's child operations and resources will remain inaccessible for the duration of the upgrade.

<Note type={"caution"}>

The startup time for some containers may exceed expected wait times and timeout, causing the upgrade to fail. The timeout period can be extended by [modifying](/update-system-properties) the `containerStartUpTimeoutSeconds` [system property](/system-properties) to a value larger than the default 1,800 seconds. If the upgrade process continues to fail after modifying the timeout period and restarting, contact [Esri technical support](https://support.esri.com/en/contact-tech-support). This only applies when installing a patch.

</Note>


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `versionManifest` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The unique ID associated with a patch or release. You can get the version manifest ID for a patch or release from the JSON view of the [Available](/check-available-updates) operation.

      ```other label=Example
      {"id": "rel_234567_2000"}
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `licenseFile` 

      (Optional)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"note"}>

      This parameter was previously deprecated at 11.0 and has been reintroduced at 11.1. Licensing information can either be provided through the `licenseFile` parameter or the `upgradeSettings` parameter.

      </Note>

      The ArcGIS Enterprise on Kubernetes license file (`.json`). This license file-which contains information regarding your user types, apps, and capabilities-is obtained from My Esri. For more information, see [Obtain an authorization file](https://enterprise-k8s.arcgis.com/en/latest/deploy/obtaining-an-authorization-file.htm). This parameter is not required if the request is for a minor release or patch-based upgrade.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `versionManifestFile` 

      (Optional)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"caution"}>

      This parameter is not currently supported.

      </Note>

      The file containing the version manifest.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `upgradeSettings` 

      (Optional; Required for version 11.0)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"note"}>

      Licensing information can either be provided through the `licenseFile` parameter or the `upgradeSettings` parameter.

      </Note>

      A JSON object containing details for release upgrade settings. These settings, retrieved from the [Get Upgrade Settings](/get-upgrade-settings) operation, must be included in the request for the upgrade to be successful. Currently, the object supports the following three upgrade settings: `updateToLatestPatch`, `licenseUpload`. These settings are applicable to ArcGIS Enterprise on Kubernetes versions 11.0 and later.

      For more information about these settings, see the [Upgrade settings](/upgrade#upgrade-settings) section below.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `volumesConfig`

      (Required for deployments using cloud-native relational data stores)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note>

      This parameter is separate from the `volumesConfig` that could previously be specified as part of the `upgradeSettings` parameter.

      </Note>

      Introduced at 11.4. A JSON object containing the storage configurations for a deployment's cloud-native relational data store, ensuring that the system has the required storage volume configuration established after the upgrade completes. The `volumesConfig` JSON object must include the following properties:

      * `provisioningType`: Specifies the type of storage provisioning type (either `DYNAMIC` or `STATIC`).
      * `component`: Indicates the component being upgraded. For cloud-native relational data stores, this value must be `REALTIONAL_UPGRADE`.
      * `size`: Defines the size of the storage volume (ex: `16Gi`).
      * `storageClass`: Specifies the storage class to use.
      * `labels`: Defines any key-value pairs used to categorize and manage the storage volumes.

      When upgrading from 11.4 to 11.5 (or later), or upgrading from one later version of ArcGIS Enterprise on Kubernetes to another, this parameter will need to be included in the request if a cloud-native relational stores have been registered with the deployment. This parameter is not applicable to environments upgrading from 11.3 to 11.4.

      ```other label=Example
      {
        "volumesConfig": [
          {
            "provisioningType": "DYNAMIC",
            "component": "REALTIONAL_UPGRADE",
            "size": "16Gi",
            "storageClass": "tz405rmnc1",
            "labels": {
              "arcgis/tier": "storage",
              "arcgis/app": "postgres"
            }
          }
        ]
      }

      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `cloudConfig`

      (Required for deployments using cloud-native relational data stores)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Introduced at 11.4. A JSON array containing the deployment's cloud storage configuration for a cloud-native relational data store, including the credentials for the service. For more information on the required properties for `cloudConfig`, see the Configure documentation's [Cloud configuration properties](configure/#cloud-configuration-properties) section.

      When upgrading from 11.4 to 11.5 (or later), or upgrading from one later version of ArcGIS Enterprise on Kubernetes to another, this parameter will need to be included in the request if a cloud-native relational stores have been registered with the deployment. This parameter is not applicable to environments upgrading from 11.3 to 11.4.

      ```other label=Example

        {
          "name": "AWS",
          "credential": {
            "type": "IAM-ROLE"
          },
          "cloudServices": [
            {
              "name": "AWS RDS for PostgreSQL",

                "credential": {
                  "type": "databaseUser",
                  "databaseUsers": {
                    "adminUser": {
                      "authType": "PASSWORD",
                      "name": "<db name>",
                      "password": "<password>"
                    }
                  }
                }
              }
            }
          ]
        }
      ]

      ```

    </StyledTableCell>
  </StyledTableRow>

### Upgrade settings

The table below outlines the currently supported settings that are required when upgrading an ArcGIS Enterprise on Kubernetes deployment.

<StyledTable headers={"Property,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `updateToLatestPatch` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      When set to `true`, patches included with a release will be automatically installed during the upgrade process. The default value of this property is `true`; the value should not be changed.

      Values: `true`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `licenseUpload` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A JSON object that stores an uploaded ArcGIS Enterprise on Kubernetes license file (`.json`). When [upgrading](/upgrade), the `value` property must be updated to reflect the [upload ID](/uploads) of an [uploaded](/upload-item) ArcGIS Enterprise on Kubernetes license file.

      ```markdown label=Example
      {
        "seqNum": 1,
        "validFileExtensions": ["json"],
        "name": "licenseUpload",
        "description": "licenseFieldDescToken",
        "id": "licenseUpload",
        "type": "File",
        "value": "caf81a83-b6a7-4810-9e7c-dbff96c6050a",
        "required": true,
        "newfield": ""
      },
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `volumesConfig` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      <Note type={"note"}>

      The `volumesConfig` upgrade setting has been deprecated at ArcGIS Enterprise on Kubernetes 11.1 and is only applicable to ArcGIS Enterprise on Kubernetes 11.0.

      </Note>

      A JSON object containing the storage configurations for persistent volumes that are required while upgrading.

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `updateToLatestPatch` 

      (Optional)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

### Example two: upgrade settings

#### Prerequisites

Before performing this operation, the list of required settings needed for the upgrade must be retrieved using the [Get Upgrade Settings](/get-upgrade-settings) operation:

```other
POST /context/admin/system/upgrades/getUpgradeSettings HTTP/1.1
Host: organization.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

upgradeId={"id": "rel_08042022_3000"}&f=pjson&token=<token>
```

For example, when upgrading an ArcGIS Enterprise on Kubernetes deployment from version 10.9.1 to 11.0, you must modify the `value` property for the `licenseUpload` JSON object to reflect the [upload ID](/uploads) of an uploaded ArcGIS Enterprise on Kubernetes license file:
