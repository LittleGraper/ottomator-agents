## Description

The `currentVersion` resource returns the current version for a deployment. When an patch or release is installed, this resource will update to reflect the information included in the update's [version object](/check-available-updates) as well as the job messages recorded during the upgrade process.

<Note type={"note"}>

When the current version of the deployment is being represented, the version and build numbers are combined. For example, if the version of the deployment is 10.9.0, and the build is 505, the current version is returned as 10.9.0.505. The version number is updated with each installed release, while the build number is updated with each update (either patch or release) installed. This representation is returned through the HTML view of the API, as well as in the upgrade job messages retained by the `currentVersion` and `upgrades` resources. For an example of these messages, see the [JSON Response example](/current-version#json-response-example) section below.

</Note>

## Example usage

The following is a sample request URL used to access the `currentVersion` resource:

```markdown
https://organization.domain.com/context/admin/system/upgrades/currentVersion?f=pjson
```
