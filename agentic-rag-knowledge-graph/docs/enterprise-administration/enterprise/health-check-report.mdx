## Description

The resource for an individual health check report returns information, including whether the organization passed the health check, the [suites](/suites) that were used to perform the health check, as well as the start time, end time, name, description, and status of each test that was performed. This resource also allows administrators to export or delete the report. Both exporting and deleting a report happen automatically when those options are chosen, requiring no additional user input.

Exporting the report will immediately redirect to a job page where administrators can poll the status of the request. A completed job will provide an upload URL that can be used to access the exported report. An exported report is packaged as a .zip file that contains CSV and JSON versions of the report.

Deleting a report will immediately result in the report being removed. No response is returned.

<Note type={"tip"}>

Multiple reports can be exported or deleted using the [Reports](/reports)-level [Export](/export-reports) and [Delete](/delete-reports) operations. Both operations require a list of run IDs, which can be gathered from an individual report resource or from the [Query](/query-reports) operation.

</Note>

This resource also provides access to the [Rename](/rename-report) operation, which allows for the report's name to be modified.

## Example usage

The following is a sample request URL used to access a specific health check report:

```other
https://organization.domain.com/context/admin/healthCheck/reports/vegcl1xm6u?f=pjson
```
