## Description

The `convertToItem` operation allows the owner of a notebook to convert a snapshot to a new notebook item.

## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `itemId`

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The item ID of the notebook with the snapshot to convert to an item.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `resourceKey`

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The resource key of the snapshot to convert. 
        
      Example:

      ```other
      resourceKey=snapshot-0f146a4ba1964327b396b9be5f1042ed.json

      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `notebookTitle`

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A title for the newly created notebook item.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample POST request for the `convertToItem` operation:


```other
POST /context/admin/notebooks/snapshots/convertToItem HTTP/1.1
Host: organization.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

itemId=2ff206ba534541428409e327d03588d1&resourceKey=snapshot-0f146a4ba1964327b396b9be5f1042ed.json&notebookTitle=snapshot1&f=pjson&token=<token>

```

## JSON Response example

```json
{
  "itemId": "7f95c2b30f4e43579bb2964bb4e352ac",
  "status": "success"
}
```
