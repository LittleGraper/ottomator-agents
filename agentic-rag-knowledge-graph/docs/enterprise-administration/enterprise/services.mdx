## Description

The `services` resource acts as the root folder and container for all subfolders and GIS services for your deployment. You can create a new subfolder by using the [Create Folder](/create-folder) operation as well as a create new GIS service by using [Create Service](/create-service).


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample request URL used to access the `services` resource:

```markdown
https://organization.domain.com/context/admin/services?f=pjson
```
