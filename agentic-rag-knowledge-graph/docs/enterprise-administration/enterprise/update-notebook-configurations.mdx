## Description

The `update` operation allows you to update notebook configurations. The update operation overwrites all existing configurations.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `configuration`

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The JSON object containing the updated [configuration](/notebook-configuration).

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default response format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

## Example usage

The following is a sample POST request for the `update` operation:


```other
POST /context/admin/notebooks/configuration/update HTTP/1.1
Host: organization.domain.com
Content-Type: application/x-www-form-urlencoded
Content-Length: []

configuration={"webSocketSize": 24}&f=pjson&token=<token>

```
