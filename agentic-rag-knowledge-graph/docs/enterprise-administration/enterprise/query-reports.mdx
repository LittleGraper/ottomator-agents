## Description

The `query` operation returns all current health check reports for the organization, returning name of the operation, the start and end time of the health check in UTC format, and whether the organization passed or failed the check, as well as the IDs associated with each report that are needed when using the [Reports](/reports)-level [Export](/export-reports) and [Delete](/delete-reports) operations. The HTML view of the response provides access to the [Report](/health-check-report) resource.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `runResult` 

      (Optional)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Filters reports by their health check result.

      Values: `passed` | `failed`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `num` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The maximum number of results to be included in the response. The default value is 1000.

      ```markdown label=Example
      num=1000
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>


## Example usage

The following is a sample request for the `query` operation:

```markdown
https://organization.domain.com/context/admin/healthCheck/reports/query?runResult=passed&num=1000&f=html
```
