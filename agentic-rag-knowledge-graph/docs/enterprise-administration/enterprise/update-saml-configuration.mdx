## Description

The `update` operation enables SAML certification when the `identityCertificateName` value for an [imported SAML identity certificate](/import-identity-certificates) is provided and SAML configuration is enabled.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `samlSecurityConfig` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A JSON object containing the `identityCertificateName` value for the imported SAML identity certificate (`samlcert`) and the enabled status of the SAML configuration. SAML will only be configured if the enabled status is set to `true`.

      ```markdown label=Example
      samlSecurityConfig={
        "enabled": true,
        "identityCertificateName": "samlcert"
      }

      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>

POST /context/admin/security/saml/update HTTP/1.1

Content-Type: application/x-www-form-urlencoded
