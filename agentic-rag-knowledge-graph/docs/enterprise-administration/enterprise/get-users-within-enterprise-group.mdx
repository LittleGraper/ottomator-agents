## Description

The `getUsersWithinEnterpriseGroup` operation returns a list of users who are assigned to the enterprise group within the enterprise user and group stores. The `filter` parameter can be used to narrow search results.

<Note type={"note"}>

This operation returns nested users if the security configuration has been configured to support nested users and groups.

</Note>


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `groupName` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Specifies the name of the enterprise group for which the user accounts need to be listed.

      ```markdown label=Example
      groupName=cartographers
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `filter` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      An optional parameter that can be used to filter search results to return specific users.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `maxCount` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Specifies the maximum number of results the client is willing to accept.

      ```markdown label=Example
      maxCount=10
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>
