## Description

The `uploadPart` operation uploads one part of an item. Before uploading parts, use the [Register](/register-uploaded-item) operation to reserve space for the item on the server. Once all the necessary parts have been uploaded to create the item using `uploadPart`, use the [Commit](/commit-upload) operation to consolidate the parts.


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `partNumber` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      An integer value associated with the part.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `partFile` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The file for the part being uploaded, processed as a multipart POST file.

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. Currently, the only supported format is `html`.

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>
