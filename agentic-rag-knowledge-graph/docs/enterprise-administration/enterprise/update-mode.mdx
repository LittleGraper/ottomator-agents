## Description

The `update` operation enables read-only mode for an ArcGIS Enterprise on Kubernetes organization. It is recommended that you enable read-only mode when upgrading a deployment to ensure that no data loss or downtime occurs. When read-only mode is enabled, users can interact with existing content, but they cannot create or modify data (such as creating a new user or updating the properties of a feature service).


## Request parameters

<StyledTable headers={"Parameter,Details"}>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `isReadOnly` 

      (Required)

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      A Boolean that specifies whether the organization is in read-only mode. Read-only mode will block requests to modify or create data, including content, users, groups, and site settings. The default value is `false`.

      Values: `true` | `false`

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `message` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      Sets a custom message to be displayed whenever an attempt to modify or update content or site settings is made through the API. If no custom message is provided, a default response is used.

      ```markdown label=Example
      message=ArcGIS Enterprise is currently in read-only mode. Contact your administrator for further assistance.
      ```

    </StyledTableCell>
  </StyledTableRow>
  <StyledTableRow>
    <StyledTableCell cellStyle="markdown">

      `f` 

    </StyledTableCell>
    <StyledTableCell cellStyle="markdown">

      The response format. The default format is `html`.

      Values: `html` | `json` | `pjson`

    </StyledTableCell>
  </StyledTableRow>
</StyledTable>
